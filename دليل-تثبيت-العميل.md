# دليل تثبيت النظام على أجهزة العملاء

## متطلبات التشغيل

### على جهاز السيرفر:
1. **تشغيل السيرفر** باستخدام الملف: `تشغيل-السيرفر.bat`
2. السيرفر سيعمل على منفذين:
   - **المنفذ 3000**: API Server
   - **المنفذ 8080**: Frontend Server
3. **الحصول على IP السيرفر**: سيظهر تلقائياً عند تشغيل السيرفر

### على أجهزة العملاء:
1. نسخ الملف `تثبيت-على-جهاز-العميل.bat` إلى جهاز العميل
2. تشغيل الملف كمسؤول (Run as Administrator) - اختياري
3. إدخال IP السيرفر عندما يُطلب منك

---

## خطوات التثبيت

### الخطوة 1: تشغيل السيرفر
```
1. افتح المجلد: c:\Users\ahmed\vs code\hr
2. قم بتشغيل: تشغيل-السيرفر.bat
3. انتظر حتى يظهر IP السيرفر (مثال: 192.168.1.100)
4. احتفظ بهذا الـ IP
```

### الخطوة 2: تثبيت على جهاز العميل
```
1. انسخ الملف "تثبيت-على-جهاز-العميل.bat" إلى جهاز العميل
2. قم بتشغيل الملف
3. أدخل IP السيرفر (مثال: 192.168.1.100)
4. انتظر حتى يكتمل التثبيت
5. سيظهر اختصار على سطح المكتب باسم "نظام الموارد البشرية"
```

### الخطوة 3: فتح النظام
```
1. اضغط مرتين على الاختصار على سطح المكتب
2. سيفتح المتصفح تلقائياً على صفحة تسجيل الدخول
3. استخدم بيانات الدخول:
   - مدير: admin / admin123
   - موظف: employee1 / emp123
```

## ما الذي يحدث عند التثبيت؟

### على جهاز العميل:
1. **إنشاء مجلد البرنامج**:
   - المسار: `%LOCALAPPDATA%\HR-System`
   - يحتوي على ملفات التشغيل والمعلومات

2. **إنشاء ملف تشغيل النظام**:
   - اسم الملف: `تشغيل-النظام.bat`
   - وظيفته: فتح النظام في المتصفح

3. **إنشاء ملف معلومات الاتصال**:
   - اسم الملف: `معلومات-الاتصال.txt`
   - يحتوي على IP السيرفر والروابط

4. **إنشاء اختصار على سطح المكتب**:
   - اسم الاختصار: "نظام الموارد البشرية"

## استكشاف الأخطاء وحلها

### المشكلة: لا يمكن الوصول إلى السيرفر

**الأسباب المحتملة:**
1. **IP السيرفر خاطئ**
   - الحل: استخدم `تحديث-اعدادات-العميل.bat` لتحديث IP

2. **السيرفر لا يعمل**
   - الحل: تأكد من تشغيل `تشغيل-السيرفر.bat` على جهاز السيرفر

3. **Firewall يمنع الاتصال**
   - الحل على السيرفر:
     ```powershell
     # تشغيل PowerShell كمسؤول
     New-NetFirewallRule -DisplayName "HR System - API" -Direction Inbound -LocalPort 3000 -Protocol TCP -Action Allow
     New-NetFirewallRule -DisplayName "HR System - Frontend" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Allow
     ```

4. **الأجهزة على شبكات مختلفة**
   - الحل: تأكد من أن السيرفر والعملاء على نفس الشبكة المحلية

### المشكلة: النظام لا يفتح

**الحل:**
1. افتح المتصفح يدوياً
2. اكتب في شريط العنوان:
   ```
   http://IP-السيرفر:8080/login.html
   ```
   مثال: `http://192.168.1.100:8080/login.html`

### المشكلة: تغير IP السيرفر

**الحل:**
1. استخدم ملف `تحديث-اعدادات-العميل.bat`
2. أدخل IP الجديد
3. سيتم تحديث كل الإعدادات تلقائياً

## معلومات إضافية

### الملفات المستخدمة:
- `تثبيت-على-جهاز-العميل.bat` - للتثبيت الأول
- `تحديث-اعدادات-العميل.bat` - لتحديث IP السيرفر
- `تشغيل-السيرفر.bat` - لتشغيل السيرفر

### موقع ملفات البرنامج على جهاز العميل:
```
%LOCALAPPDATA%\HR-System\
├── تشغيل-النظام.bat
└── معلومات-الاتصال.txt
```

### للوصول السريع لمجلد البرنامج:
1. اضغط `Win + R`
2. اكتب: `%LOCALAPPDATA%\HR-System`
3. اضغط Enter

## الأمان

- النظام مصمم للعمل على الشبكة المحلية فقط
- لا تشارك بيانات الدخول مع غير المصرح لهم
- تأكد من تشغيل الـ Firewall على السيرفر
- غيّر كلمات المرور الافتراضية بعد أول تسجيل دخول

## الدعم الفني

في حالة وجود مشاكل:
1. راجع ملف `معلومات-الاتصال.txt` في مجلد البرنامج
2. تأكد من تشغيل السيرفر
3. تحقق من اتصال الشبكة
4. راجع قسم استكشاف الأخطاء في هذا الدليل
